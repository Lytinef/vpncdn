# Подписки: жизненный цикл

## Тарифы

| Код        | Название  | Устройств | Цена/мес |
|------------|-----------|-----------|----------|
| `start`    | Старт     | 1         | 149 ₽    |
| `standard` | Стандарт  | 3         | 249 ₽    |
| `family`   | Семейная  | 6         | 349 ₽    |

Период — 30 дней. Цены хранятся в копейках.

## Статусы

| Статус      | Доступ | Описание                                                |
|-------------|--------|---------------------------------------------------------|
| `pending`   | нет    | создана, ждёт первого успешного платежа                 |
| `active`    | да     | активна                                                 |
| `past_due`  | да*    | продление не прошло, grace до конца периода             |
| `canceled`  | да     | отменена пользователем, доступ до `currentPeriodEnd`    |
| `expired`   | нет    | период закончился, продления не будет                   |

\* доступ сохраняется, пока `currentPeriodEnd` в будущем.

## Покупка

1. `POST /payments/checkout {planCode}` → создаётся `pending`-подписка и платёж в YooKassa
   с `save_payment_method=true`; возвращается `confirmationUrl`.
2. Клиент оплачивает (WebView). YooKassa присылает webhook → backend перепроверяет статус.
3. При `succeeded`: сохраняется карта, подписка → `active`, период = now…now+30д.

## Отмена (в конце периода)

`POST /subscription/cancel` → `cancelAtPeriodEnd=true`, `autoRenew=false`,
статус `canceled`. **Доступ сохраняется до `currentPeriodEnd`**, деньги не возвращаются.
Возобновление до конца периода — `POST /subscription/resume`.

## Смена тарифа (со следующего периода)

`POST /subscription/change-plan {planCode}` → заполняется `nextPlan`.
Текущий период не меняется; при продлении подписка переходит на новый тариф,
и автосписание идёт уже по новой цене. Повторный выбор текущего тарифа отменяет смену.

## Автосписание (рекуррент)

- Cron (`RenewalScheduler`, раз в час) находит подписки с `autoRenew=true`,
  у которых `currentPeriodEnd <= now`, и ставит задачу в очередь BullMQ.
- `RenewalProcessor` → `PaymentsService.chargeRenewal`:
  - сумма = цена `nextPlan ?? plan`;
  - рекуррентный платёж по сохранённой карте;
  - при успехе — `renewAfterPayment` (применяет смену тарифа, продлевает период);
  - при неуспехе — `past_due` + повторные попытки (backoff). 
- Подписки `canceled`/`past_due` с истёкшим периодом переводятся в `expired`.

## Лимит устройств

Активная подписка задаёт `deviceLimit`. Регистрация нового устройства сверх лимита
запрещена (нужно удалить устройство или сменить тариф). Понижение тарифа применяется
со следующего периода.
